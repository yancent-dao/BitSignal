# BitSignal 产品设计蓝图

**事件驱动情报 Agent 网络 · 接入 x402 · 上架 AgentVerse 变现**

版本 v0.2 ｜ 2026-06-18 ｜ 目标读者：部署了量化交易 Agent、需要"某领域实时事件与影响信号"的用户与 Agent

> v0.2 重大调整：**不做实时行情分发**（已有成熟 API、且涉及交易所授权）。改做**事件驱动情报**——监听特朗普推文、白宫新闻、美联储发言等高影响力信源，毫秒级检测并输出"事件 → 受影响标的 → 方向/强度/置信度"的机器可读信号。同时按用户要求**暂不做回测模块**。

---

## 0. 一句话定位

**当一个能撬动市场的人或机构发声时，第一时间把它翻译成机器可读的交易信号。** 我们监听高影响力信源（特朗普 Truth Social + X、白宫、美联储、监管机构……），在事件发生的瞬间检测、分类、映射到受影响资产，输出结构化信号——量化 Agent 通过 x402 按次或订阅付费获取，无需订阅合同、无需 API key。

我们卖的不是"原始行情"（那是红海，已有成熟 API），而是**"事件 → 影响"的实时翻译**。

---

## 1. 为什么是这个方向（机会判断）

**需求已被验证，且在高速增长。** 新闻聚合与事件驱动交易预警市场预计从 2026 年的 $26.2 亿增长到 2034 年的 $69 亿（CAGR 12.9%）。学术研究证实：有信息量的特朗普推文会引起美元升值并改变日内波动率，效果在控制宏观公告后依然稳健。市场对单条高影响力信息的反应通常在发布后 30 秒到 3 分钟内发生——**这正是事件信号的 alpha 窗口**。

**已有玩家证明了付费意愿，但留下了关键空档。** TweetStream 提供市场异动帖子的 WebSocket 流（含可选的特朗普 Truth Social 源），TrumpBot 在特朗普发帖时 webhook 推送，部分服务能在 5 秒内送达。但这些产品几乎都停留在"**推原文 / 简单提醒**"层面：

- 它们告诉你"特朗普发了一条帖"，但不告诉你"**这条帖利空哪些股、利好哪些币、置信度多高、多快传导**"。
- 它们几乎都是**面向人**的提醒，**没有人做成 x402 原生、可被其他量化 Agent 自动发现和调用的 A2A 信号源**。

**我们的卡位**：(1) 输出结构化的"事件 → 影响标的 → 方向 / 强度 / 置信度"，而非原文；(2) 机器可读、x402 按次付费，让别人的量化 Agent 在运行时自动调用；(3) 全信源 + 跨市场（美股 / 美元 / 加密 / 大宗），不局限单一人物。

**顺带解决了上一版的最大风险**——事件信号的原料是公开言论与政府发布，**几乎不涉及数据授权**，护城河落在"影响映射知识库 + 速度 + 结构化"，而不是谁都能拿到的原始 feed。

## 2. 产品总体形态：四层 Agent 架构

从"信源 → 可付费信号"的事件流水线，分四层。对外只暴露一个网关 Agent，内部是一组各司其职的微服务 Agent。

```
┌─────────────────────────────────────────────────────────────┐
│  L3 交付/网关层   Gateway & Router Agent（AgentVerse 上的门面） │
│     · Pull：x402 按次查询"最近影响X资产的事件"                   │
│     · Push：webhook / 流式订阅，事件触发毫秒级推送              │
│     · 处理 x402 402挑战→USDC结算→回执  · 计量/限流/延迟分级     │
├─────────────────────────────────────────────────────────────┤
│  L2 分析与信号层   事件→信号流水线（7 个能力型 Agent）          │
│     检测去噪 · 分类抽取 · 影响映射 · 方向强度评分               │
│     语气/新颖度 · 紧急度/时效 · 信号合成与跨市场传导            │
├─────────────────────────────────────────────────────────────┤
│  L1 信源采集层   6–7 个"监听型"Agent（按信源域×接入方式划分）   │
│     政要/意见领袖 · 政府政策 · 监管/制裁 · 宏观数据发布         │
│     企业事件 · 全球新闻流 · 加密专属事件                        │
│     → 统一归一到 Event Schema                                  │
├─────────────────────────────────────────────────────────────┤
│  L0 支付与身份层   x402 钱包 · USDC(Base) · AgentVerse 注册     │
└─────────────────────────────────────────────────────────────┘
```

**核心设计原则**：信源层按"信源域 × 接入方式"拆（社交流、政府 RSS、申报系统、新闻线接入方式完全不同）；分析层按"分析能力"拆（同一套能力复用到所有信源）；对外只一个网关，把发现、计费、推送收口到一处。**整条链路为低延迟优化**——事件检测到信号产出尽量压在秒级。

## 3. L1 信源采集层：监听哪些信源、拆几个 Agent

**结论：全量覆盖约 6–7 个监听 Agent。** 按"共享同一信源域 + 同一接入方式"归并。每个 Agent 把抓到的内容归一到统一 **Event Schema** 后推给分析层。

| # | 监听 Agent | 监听什么 | 接入方式 / 来源 |
|---|-----------|---------|----------------|
| S1 | 政要/意见领袖 | **特朗普（Truth Social + X）**、各国元首、马斯克、关键政客的实时发帖 | 社交流式 API、Truth Social 抓取、第三方转发流 |
| S2 | 政府/政策机构 | **白宫**（行政令、新闻发布会、声明）、**美联储/FOMC**、财政部、各国央行（ECB/英央行/日央行/PBoC） | 官方 RSS、新闻稿、官网监听 |
| S3 | 监管/立法/制裁 | SEC 执法、OFAC 制裁名单、CFTC、国会立法动向 | EDGAR、OFAC、Congress.gov（多免费） |
| S4 | 宏观数据发布 | 经济日历 + 实际数据（CPI、非农、利率决议、GDP）——"已知时间、未知内容"类事件 | FRED、官方发布、经济日历源 |
| S5 | 企业事件 | 8-K、财报、并购、业绩指引、CEO 公开表态 | EDGAR、PR 新闻线、财报日历 |
| S6 | 全球财经新闻流 | 多语种突发财经 / 地缘新闻 | 新闻 API、RSS、通讯社 |
| S7 | 加密专属事件 | 交易所上/下币公告、协议治理、基金会动作、加密监管新闻 | 交易所公告、治理论坛、链上信号 |

> **统一 Event Schema（采集层输出）**：`{event_id, 抓取时间戳, 信源, 发声主体, 原文/摘要, 语言, 链接}`。采集 Agent 只做"快速抓取 + 归一 + 去重初筛"，不做分析——保证低延迟。MVP 阶段只上 S1（特朗普）+ S2（白宫/美联储）即可起步。

## 4. L2 分析与信号层：可以拆出哪些 Agent

事件 → 信号是核心价值，拆成 7 个能力型 Agent，组成一条流水线。每个能力对所有信源复用。

1. **事件检测/去噪 Agent** — 这是不是一个"真事件"？去重、识别新颖性（不是转发/重复/旧闻）、过滤噪声。**速度最关键的一环**，决定整体延迟。
2. **事件分类/实体抽取 Agent** — 判定事件类型（关税 / 货币政策 / 制裁 / 并购 / 监管……），抽取实体：谁、对谁、隐含哪些标的、板块、ticker、token。
3. **影响映射 Agent** — 把事件映射到受影响资产/板块。这是**护城河**：靠一个"主体 / 话题 → 资产联动"知识库。例：特朗普关税帖 → 相关板块股、美元、特定商品；美联储偏鹰 → 利率、股指、加密。
4. **方向与强度评分 Agent** — 对每个受影响标的给：看多/看空 + 预期幅度 + 置信度。
5. **语气/新颖度 Agent** — 语气强弱、是否超出市场预期（同样的话，意外说出来冲击更大）。
6. **紧急度/时效 Agent** — 多快传导到市场、信号的时间衰减（30 秒级 vs 数小时级）。
7. **信号合成与跨市场传导 Agent** — 打包成结构化预警，并推演二阶效应（如：事件 → 美元走强 → 新兴市场 → 加密）。

> **暂不做回测模块**（按你要求）。但没有回测时仍需建立信任——替代方案：**公开实时事件日志 + 命中展示**（"此信号发出 N 分钟后该标的如何变动"的透明记录），让买家自己看到准确性，成本低且即时。后续若要做回测，作为独立 Agent 补上即可。

## 5. 产品与信号形态：卖什么、什么样

**核心交付物 = Event-Impact Signal（事件影响信号）**，结构化、机器可读：

```json
{
  "event_id": "...",
  "detected_at": "2026-06-18T14:32:01.120Z",
  "source": "Truth Social",
  "actor": "Donald Trump",
  "event_type": "tariff_threat",
  "summary": "威胁对进口汽车加征关税",
  "entities": ["auto sector", "EU", "USD"],
  "impacts": [
    {"symbol": "汽车板块", "market": "US equities", "direction": "bearish", "magnitude": "medium", "confidence": 0.78},
    {"symbol": "USD", "market": "FX", "direction": "bullish", "magnitude": "low", "confidence": 0.65},
    {"symbol": "BTC", "market": "crypto", "direction": "neutral", "confidence": 0.4}
  ],
  "sentiment": "aggressive",
  "novelty": "high",
  "urgency": "immediate",
  "expected_time_to_impact": "<3min",
  "rationale": "..."
}
```

**两种交付形态**：
- **Push（推）**：webhook / 流式订阅，事件触发毫秒级推送——延迟即 alpha，溢价最高。
- **Pull（拉）**：x402 按次查询，如"最近 1 小时影响美股科技板块的事件""特朗普今天关于关税的所有表态及影响"。

## 6. 分层定价（x402 上卖什么）

单位 USDC（Base 链）。在"加工程度"和"延迟"两个维度上分层。

| 档位 | 内容 | 参考定价 | 说明 |
|------|------|---------|------|
| **T0 归一化事件流** | 原文 + 元数据（谁、何时、何源） | $0.005–0.01/条 | 走量，对标现有"推原文"玩家但机器可读 |
| **T1 事件 + 分类抽取** | 加事件类型、实体、涉及标的 | $0.02–0.05/条 | — |
| **T2 事件 + 影响映射 + 评分** | 完整 Event-Impact Signal（方向/强度/置信度） | $0.10–0.30/条 | **主力高毛利**，对标 Nansen 专有信号 $0.05 但价值更高 |
| **T3 高影响力专属即时频道** | 特朗普 / 美联储 / 白宫专属频道 + webhook 毫秒推送 | $0.30–1.0/条 或包月订阅 | **延迟 = alpha**，溢价最高 |

**延迟分级**：< 5 秒即时推送走 T3 溢价；分钟级延迟档便宜走量。**量价机制**：高频订阅走预付额度 / 包月，x402 会话扣费。**A2A 分发**：信号 Agent 本身可被其他量化 Agent 在 AgentVerse 上自动发现并调用，是复利增长点。

> 算账直觉：一个事件驱动量化 Agent 订阅特朗普 + 美联储 T3 频道，假设月费 $300–800；几十个这样的订阅 + per-call 长尾，就能撑起稳定现金流。关键是把"被发现 → 首次调用 → 持续订阅"漏斗跑通。

## 7. L3 网关层与 x402 / AgentVerse 接入

**网关/路由 Agent 是唯一对外门面，也是变现收口点：**

1. 通过 AgentVerse 注册（uAgents 框架，Python 3.10–3.13），用 Agent Chat Protocol 接入 ASI:One，挂关键词提升可发现性：`event signals`、`political catalyst`、`Fed alerts`、`Trump signal`、`crypto news signal`、`macro event`。
2. 同时支持 **Pull**（x402 按次查询，返回 402 报价 → USDC 支付 → 回执 → 结果）和 **Push**（订阅方预付额度，事件触发即推 webhook，按条扣费）。
3. 负责计量、限流、按档/按延迟计费、用量日志（链上回执即审计轨迹）。

**部署**：uAgents 轻量 Python 库，监听/分析 Agent 部署在 Render / Railway / GCP / AWS，通过 mailbox/proxy 注册到 AgentVerse。

**变现通道优先级**：(1) x402 按次 + 订阅（USDC on Base）= 主力；(2) T3 专属频道包月；(3) A2A 自动分发；(4) AgentVerse Agent Launch 代币机制——投机性强，**等现金流跑通后再评估，勿本末倒置**。

## 8. 分阶段路线图

**Phase 0 — MVP（约 3–5 周，单人，全用免费/低价信源）**
目标：跑通"高影响力事件 → 结构化信号 → x402 付费"最小闭环。
- 信源：S1 特朗普（Truth Social + X）+ S2 白宫/美联储官方源
- 分析：检测去噪 → 分类抽取 → 影响映射 → 方向强度评分 → 打包（最小链）
- 影响资产先聚焦：美股板块 + 美元 + BTC（信号最强、映射最清晰）
- 网关：1 个 x402 网关，支持 per-call + 一个 webhook 推送频道
- 上架 AgentVerse，挂 T0/T2 端点 + 1 个 T3 特朗普频道
- **验收：拿到第一批真实付费调用/订阅，验证定价与延迟。**

**Phase 1 — 加深政策/监管 + 跨市场**
加 S3 监管/制裁、S4 宏观数据发布；持续扩充影响映射知识库；上线跨市场传导 Agent；公开实时命中日志建立信任。

**Phase 2 — 扩信源 + 扩资产**
加 S5 企业事件、S6 全球多语种新闻、S7 加密专属事件；影响资产扩到欧股/亚太/大宗/更多币种。

**Phase 3 — 全球化 + A2A 生态**
加多国政要/央行信源；铺多个 T3 专属频道（各国元首、各大央行）；推动信号 Agent 被其他量化 Agent 自动调用，形成网络效应。

## 9. 风险与应对

- **信源接入风险（头号）**：Truth Social / X 的 API 政策、限流与成本（X API 较贵）。应对：多路冗余（官方 API + 第三方转发流 + 抓取），合规采集，对单点依赖做降级预案。
- **延迟竞争**：已有玩家能做到 < 5 秒。我们**不靠纯比快赢**（纯拼速度拼不过 HFT 自建），靠"结构化 + 影响映射 + 机器可读 + A2A 可调用"的综合价值赢；同时延迟也要尽量压到秒级以保持竞争力。
- **影响映射准确性**：错误映射伤信誉。应对：初期映射保守、始终附置信度、高价 T3 信号可加人工复核；用透明命中日志让买家自行评估。
- **法律风险**：引用/转述他人言论注意版权与诽谤。应对：**只做事实陈述 + 影响分析，绝不编造引文**；对信号附免责声明。
- **合规定位**：我们卖"事件与信号"，不是"投资建议"，不替用户下单、不做个性化荐股。
- **信任（无回测）**：暂不做回测的情况下，用公开实时事件日志 + 命中展示替代；后续可作为独立 Agent 补上。
