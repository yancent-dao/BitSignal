# BitSignal-TrumpSignal · 产品需求

**Agent 名称**：TrumpSignal（事件驱动情报 Agent，以特朗普发帖为核心信号源）
**所属产品**：BitSignal
**阶段**：Phase 0 / MVP
**版本**：v0.1 ｜ 2026-06-18

> 本文档只描述**做什么**（产品需求与验收标准）。**怎么做**——技术选型、接口/Schema 定义、模块拆分、任务清单——由 Claude Code 在编码时决定。
> 设计背景见 `../docs/产品设计蓝图.md`。

---

## 1. 产品目标

TrumpSignal 监听以特朗普为核心的少数高影响力信源，在能撬动市场的言论/事件发生的瞬间，把它翻译成**机器可读的交易信号**（受影响标的 + 方向 + 强度 + 置信度），供量化交易 Agent 通过 x402 按次或订阅付费获取。

MVP 的唯一目标：**跑通"高影响力事件 → 结构化信号 → x402 付费获取"的端到端最小闭环，并拿到第一批真实付费调用。**

---

## 2. 范围（MVP 只做这些）

**监听信源**（仅这三类）：
- 特朗普的 Truth Social 与 X（Twitter）实时发帖。
- 白宫官方发布（声明、行政令、新闻发布会通稿）。
- 美联储 / FOMC 官方发布与公开讲话。

**影响资产范围**（信号只覆盖这些，映射最清晰）：
- 美股主要板块（如汽车、能源、科技、银行等行业层面，不必到个股）。
- 美元（USD / DXY）。
- 比特币（BTC）。

**明确不做**（Out of scope）：
- 不做实时行情分发（已有成熟 API）。
- 不做回测模块（后续阶段再议）。
- 不替用户下单、不做个性化荐股。
- 不做个股级精确预测（先到板块/资产类别即可）。

## 3. 功能需求

### 3.1 信源监听
- 必须实时监听上述三类信源，新内容出现后应在**数秒内**被系统捕获。
- 对每条捕获内容记录：来源、发声主体、原文、时间戳、原始链接。
- 对重复/转发/旧闻要能去重，不重复产出信号。

### 3.2 事件识别与分析
对每条捕获内容，系统需依次完成：
- **判定是否为"真事件"**：过滤闲聊、问候、无市场含义的内容；只有具备市场含义的才继续。
- **事件分类**：判定事件类型（如关税、货币政策、制裁、监管、人事、地缘等）。
- **实体抽取**：识别涉及的主体、国家、行业、资产。
- **影响映射**：把事件映射到第 2 节范围内的受影响资产/板块。
- **方向与强度评分**：对每个受影响标的给出方向（看多/看空/中性）、预期幅度（高/中/低）、置信度（0–1）。
- **辅助判断**：语气强弱、是否超出市场预期（新颖度）、预计多快传导到市场。

### 3.3 输出信号
每条信号至少包含以下信息（字段命名与数据结构由编码时定）：
- 事件标识、检测时间戳、来源、发声主体。
- 事件类型、原文摘要。
- 受影响标的列表，每个含：标的、所属市场、方向、强度、置信度。
- 整体语气、新颖度、紧急度、预计传导时间。
- 简短的判断理由（rationale），便于买家理解信号依据。

### 3.4 交付方式（两种都要）
- **Pull（按次查询）**：买家 Agent 可查询，如"最近 1 小时影响美股能源板块的事件"、"特朗普今天关于关税的表态及影响"。
- **Push（订阅推送）**：买家可订阅某信源频道（如"特朗普频道"），事件触发时主动推送（webhook 形式），延迟尽量压到秒级。

### 3.5 支付与变现
- 通过 **x402 协议**收款，结算用 **USDC（Base 链）**，无需买家注册或订阅合同即可按次付费。
- 支持分档计价（MVP 可先实现其中 2 档）：
  - 低档：归一化事件流（原文 + 元数据）。
  - 主力档：完整影响信号（含方向/强度/置信度）。
  - 高档：高影响力专属频道 + 秒级 webhook 推送。
- 需记录每次调用/推送的计量与扣费日志（链上回执即可作为凭证）。

### 3.6 上架与可发现
- 作为 uAgent 注册到 **AgentVerse**，接入 ASI:One，使其可被其他 Agent 搜索与调用。
- 挂好搜索关键词，如：`event signals`、`political catalyst`、`Fed alerts`、`Trump signal`、`macro event`。

## 4. 非功能需求

- **延迟**：从信源出现内容到产出可获取信号，目标端到端尽量压在秒级（不追求 HFT 微秒级）。
- **可靠性**：信源接入要有冗余与降级预案（单一来源限流/失效时不致整体瘫痪）。
- **可观测**：保留完整的实时事件日志（事件 → 信号 → 发出时间），既用于排障，也用于向买家展示命中情况（替代回测建立信任）。
- **合规**：只做事实陈述与影响分析，**绝不编造引文**；信号输出附免责声明，明确"非投资建议"。
- **成本意识**：MVP 优先使用免费/低价信源接入方式（X API 较贵，注意成本与限流）。

---

## 5. 验收标准（MVP 通过的判定）

1. 当特朗普发出一条有市场含义的帖子（如关税威胁），系统能在**数秒内**捕获，并产出一条包含"受影响板块/USD/BTC + 方向 + 强度 + 置信度 + 理由"的结构化信号。
2. 白宫、美联储信源的有市场含义事件同样能产出信号；无市场含义的内容被正确过滤、不产出信号。
3. 一个外部调用方能通过 **x402 用 USDC 付费**后，成功**按次查询**到信号；也能**订阅频道**并在事件触发时收到 **webhook 推送**。
4. Agent 已注册到 AgentVerse，可被搜索到并完成一次端到端的付费调用。
5. 系统留有可查的实时事件日志。

## 6. 给 Claude Code 的提示

- 在 `trumpsignal/` 目录内编码；技术栈、目录结构、接口/Schema、模块拆分、任务顺序都由你决定，但需对齐本文档的功能与验收标准。
- AgentVerse 部署用 uAgents（Python 3.10–3.13）；x402 结算用 USDC on Base。建议先查最新官方文档确认 API 细节。
- 建议从最小可跑链路做起：单信源（特朗普）→ 检测/分类/映射 → 输出信号 → x402 按次查询，跑通后再加白宫/美联储与 webhook 推送。
